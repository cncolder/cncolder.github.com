Hello F# 我的第一个AutoCAD+F#程序 

#!fsharp
// 使用 F# 缩略语法
#light

// 声明独有的命令空间和类
module FsTest.DrawLine

// 指定包含文件夹
#I @"D:\\AutoCAD 2008"

// 引用托管
#r "acdbmgd.dll"
#r "acmgd.dll"

open Autodesk.AutoCAD.Runtime
open Autodesk.AutoCAD.ApplicationServices
open Autodesk.AutoCAD.DatabaseServices
open Autodesk.AutoCAD.Geometry

// 声明命令
[<commandmethod  (?DRAWLINE?)>]
let DrawLine () =
    let line = new Line(new Point3d(0.0, 0.0, 0.0), new Point3d(3333.0, 3333.0, 0.0))
    let db = Application.DocumentManager.MdiActiveDocument.Database
    use trans = db.TransactionManager.StartTransaction()
    let bt = trans.GetObject(db.BlockTableId, OpenMode.ForRead):?> BlockTable
    let btr = trans.GetObject(bt.[BlockTableRecord.ModelSpace], OpenMode.ForWrite):?> BlockTableRecord
    btr.AppendEntity(line) |> ignore
    trans.AddNewlyCreatedDBObject(line, true)
    trans.Commit()

#!csharp
// 下面是才鸟老大在 DotNetARX 帮助文件中, 第一页 Geting Started 的第一段 C# 示例.
// 对比一下还是蛮像的嘛~_~
Line line = new Line(new Point3d(0.0, 0.0, 0.0), new Point3d(3333.0, 3333.0, 0.0));
Database db = Application.DocumentManager.MdiActiveDocument.Database;
using (Transaction trans = db.TransactionManager.StartTransaction())
{
    BlockTable bt = (BlockTable)trans.GetObject(db.BlockTableId, OpenMode.ForRead, false);
    BlockTableRecord btr = (BlockTableRecord)trans.GetObject(bt[BlockTableRecord.ModelSpace], OpenMode.ForWrite, false);
    btr.AppendEntity(line);
    trans.AddNewlyCreatedDBObject(line, true);
    trans.Commit();
}

这不是斯巴达

是的, 这不是斯巴达. 那是什么呢? 反正不是斯巴达…

相信很多人都看过三百斯巴达勇士这部电影, 如果你还没有, 那么赶紧去补课吧, 相信我不会错, 这绝对是去年年末一部非常值得观看的电影.

电影讲述的是公元前480年, 波斯王薛西斯率50万铁骑攻向希腊. 斯巴达国王率300人对抗数10万波斯大军的故事.

那么第二幅图片是什么? 这不是斯巴达. 是的你没有看错, 这就是电影的名字.

其实了解这部影片并不难, 你只需闭上眼睛去回忆一下三百斯巴达勇士, 就能得到大至的故事框架以及所遵循的发展节奏…

不过你需要对原作有所了解, 并且还要对欧美文化有所了解, 才能真正领会电影里处处让人大跌眼晴的恶搞.

BTW: 看第一部电影的时候别太激动, 看第二部电影的时候别吃零食.

冰灾

由于罕见暴雪冻雨灾害, 造成电力铁塔覆冰, 火车飞机停运, 高速公路拥堵, 并导致数以万计的春运旅客滞留在了回家的路上.

现代社会人类对于科技的依赖性加重, 一旦发生自然灾害, 无法体现自身的本能. 试想一下, 如果这场暴风雪发生在数百年以前, 或许根本不是灾害, 而是百年奇景, 吸引文人舞文弄墨.

而我们现代的人类却完全笑不出来了! 看看我们丢失了哪些本能!

视力: 停电后我们会处在一片漆黑当中寸步难行, 蜡烛对我们来说实在是太过微弱, 在光污染如此严重的今天, 我们早已失去了夜间观察的能力. 要知道古时候点着煤油灯就算是”灯火通明”了.
取暖: 如今人们的取暖方式, 北方以煤为主, 南方以电为主, 这次的灾害正是由于天气影响电力, 电力影响运输, 运输影响煤炭, 煤炭又影响电力的恶性循环造成的, 现如今, 你一定不会指望有谁能在没有煤和电的情况下取暖, 对于我们的先人, 劈柴生火是再平常不过的事儿了.
交通: 交通的发达让世界变得越来越小, 我们正不断的提高运力和速度, 让人们可以更加容易的远行, 现在的孩子几乎不会有步行数公里的经历, 便捷的交通工具让他们对红军长征二万五千里没有任何概念.

但愿用失去的这些换来科技的繁荣, 是值得的…

使用 Firebug 查找 Wordpress 缓慢的原因

如果你使用Firefox+Wordpress, 那么我建议你下载一个Firebug插件.

想知道原因? 看图说话:

一直以来, 我的WP读取速度感觉很不爽, 10s+

搞的我都不好意思和别人说我的域名,

还曾经打电话问客服加大空间能不能提速, 空间商的网管人不错, 他告诉我容量和速度无关.

54个请求, 用时9.02秒. 偶滴个神呀!

我当然不愿意怀疑WP的效率, 于是找原因:

imghover.js这个家伙就占据了5.08秒? 而且是因为404错误?

马上找原因, 原来是皮肤里调用的这个文件, 而这个文件其实是不存在的…

imghover.js 用时5.08秒, 404错误!

拉出去枪毙, 情况马上就在有改观:

少了一个请求, 速度提高五倍! 爽!

53个请求, 用时1.69秒. 神终于显灵了!
